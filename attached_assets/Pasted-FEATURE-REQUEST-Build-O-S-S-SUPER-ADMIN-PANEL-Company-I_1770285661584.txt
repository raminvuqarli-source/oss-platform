FEATURE REQUEST: Build O.S.S SUPER ADMIN PANEL (Company Internal Only) + GET QUOTE CRM

We need a separate internal admin panel for the O.S.S team (company admins), NOT for hotel customers.

This “O.S.S Super Admin Panel” must be completely isolated from hotel dashboards and must only be accessible to O.S.S internal admins.

------------------------------------------------------------
1) PURPOSE
------------------------------------------------------------
- O.S.S team must manage all incoming GET QUOTE leads and customers.
- Hotel customers MUST NOT see this panel or its data.

------------------------------------------------------------
2) ACCESS & SECURITY (CRITICAL)
------------------------------------------------------------
A) Create a new role:
- OSS_SUPER_ADMIN

B) Access rules:
- Only users with role OSS_SUPER_ADMIN can access the Super Admin panel routes/API.
- Hotel Admin / Reception / Guest must be blocked (frontend + backend).
- If unauthorized user tries, return 403 and show a friendly “Access denied” page.

C) Separate routes:
- Use a dedicated route namespace, e.g. /oss-admin (preferred) or /super-admin
Examples:
- /oss-admin/login
- /oss-admin/dashboard
- /oss-admin/quotes
- /oss-admin/hotels
- /oss-admin/users

D) Login:
- Either reuse existing auth but enforce role OSS_SUPER_ADMIN,
  OR create a separate login screen that still uses the same auth service but checks the role.
- After login, redirect to /oss-admin/dashboard.

------------------------------------------------------------
3) SUPER ADMIN PANEL SECTIONS (MVP)
------------------------------------------------------------
A) Dashboard (Overview)
- KPIs:
  • New Quote Requests (last 7 days)
  • Total Quote Requests
  • Requests by Country
  • Requests by Status (NEW/PROCESSING/CLOSED)
  • Conversion pipeline counts (optional)
- Simple charts/tables (keep clean and fast)

B) Quote Requests (CRM Core)
- List view table columns:
  • Date
  • Hotel Name
  • Country / City
  • Contact Person
  • Phone
  • Email
  • Total Rooms
  • Expected Smart Rooms
  • Status (NEW / CONTACTED / DEMO_SCHEDULED / NEGOTIATION / CLOSED_WON / CLOSED_LOST)
  • Assigned To (OSS team member)
- Filters:
  • Status
  • Country
  • Date range
  • Search (hotel name / email / phone)
- Detail view:
  • Full submitted form data
  • Interested features
  • Preferred contact hours + timezone
  • Source page (Landing/Login/Admin)
  • Language
  • CreatedAt
  • Activity timeline
  • Internal Notes (multi-note)
  • Change Status
  • Assign to team member
  • “Mark as contacted” quick action

C) Hotels (Customer Directory) – Basic
- Create a simple directory of hotels that submitted quotes:
  • Hotel Name, Country, City, Email, Phone, CreatedAt, Status
- Ability to link quote request to a hotel record.

D) OSS Team Users
- Manage OSS_SUPER_ADMIN users only:
  • list users with OSS_SUPER_ADMIN
  • create / deactivate (if you already have user management)
  • assign quotes to users
